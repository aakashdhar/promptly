# Spec Review — FEATURE-009 History Panel — add-feature
> Date: 2026-04-19 | Trigger: add-feature
> Documents: FEATURE_SPEC.md + SPEC.md + ARCHITECTURE.md + CODEBASE.md

## Findings

### P0-001 · resize-window width hardcode breaks HISTORY width
- Issue: resize-window handler uses `win.setSize(520, height, true)` — hardcoded 520.
  openHistory() calls resizeWindowWidth(680) then transition(HISTORY) which calls
  resizeWindow(420) → resets width back to 520. HISTORY always renders at 520px.
- Fix applied: HIST-004 updated to change resize-window to `const [currentWidth] = win.getSize(); win.setSize(currentWidth, height, true)`
- Status: ✅ Fixed in FEATURE_TASKS.md

### P1-001 · ⌘H guard missing HISTORY state exclusion
- Issue: `stateRef.current !== STATES.RECORDING` guard passes in HISTORY state.
  ⌘H while in HISTORY re-calls openHistory(), overwrites prevStateRef with HISTORY.
  Done/Escape then returns to HISTORY instead of the original prior state.
- Fix applied: Guard updated to also exclude HISTORY state in HIST-003.
- Status: ✅ Fixed in FEATURE_TASKS.md

### P2 · saveToHistory signature change not noted in HIST-001
- Issue: Inline App.jsx version uses positional args; new util uses object arg.
  Agent might export positional version by mistake.
- Fix applied: HIST-001 updated with explicit signature warning note.
- Status: ✅ Fixed in FEATURE_TASKS.md

## Verdict
> P0: 1 (fixed) · P1: 1 (fixed) · P2: 1 (fixed)
